## What to build

Add the `<owner>-<repo>.md` **Telegram document delivery** layer in front of the summary message from #68. The document is a rendered markdown report of the full `repo_analysis` JSON — Tagline, Tech Stack, For Developers, For Education sections — sent via Telegram's `sendDocument` before the summary text message.

Behavior:

- New helper `render_repo_markdown(analysis: dict, bundle: dict) -> str` colocated in `processors/repo.py` (or a sibling `_render.py` if it grows past ~60 lines). Output matches spec **§Markdown document template**:

  ```markdown
  # {owner}/{repo}

  > {tagline}

  ⭐ {stars:,}  🔀 {forks:,}  💻 {language}  📅 {N days ago}

  {## ⚠️ Archived — no longer maintained\n if archived}
  {## ℹ️ No README detected — analysis is shallower than usual\n if no_readme}

  ## Tech Stack
  {bullet list of tech_stack}

  ## 🛠 For Developers

  ### Project Ideas
  {bullet list of project_ideas}

  ### When to Use
  {when_to_use prose}

  ### Avoid When
  {avoid_when prose}

  ## 🎓 For Education

  ### Concepts Taught
  {bullet list of concepts_taught}

  ### Prerequisites
  {bullet list of prerequisites}

  ### Curriculum Hooks
  {for each hook → "- **{concept}**{ — `{file_pointer}` if set}\n  {why}"}

  ---

  🔗 [{repo_url}]({repo_url})
  _Generated by vig at {timestamp}_
  ```

- `processors.repo.run` calls `telegram.sender.send_document(blob, filename=f"{owner}-{repo}.md")` BEFORE the summary text message. Filename sanitization mirrors article's pattern (strip non-`[a-zA-Z0-9 \-_]`, collapse whitespace to single space, truncate to 80 chars). Fallback filename: `{job_id}.md` if sanitization empties the owner/repo string.
- The summary message + Freestyle button (from #68) still fires after the document. Both succeed independently — if `send_document` fails (rare network blip), the summary still tries and the job still completes.
- No new flag or column needed; document is rendered from the in-memory analysis JSON and discarded after send.

Reference spec: **§Design Decisions** (#15, #16), **§Markdown document template**, **§Telegram document filename**.

## Acceptance criteria

- [ ] `render_repo_markdown(analysis, bundle)` produces a markdown string matching the spec'd structure; section headings render exactly (`## Tech Stack`, `## 🛠 For Developers`, etc.).
- [ ] When `bundle["metadata"]["archived"]` is True, the archived warning H2 is included in the document.
- [ ] When `bundle["no_readme"]` is True, the no-README info H2 is included.
- [ ] `curriculum_hooks` items with `file_pointer=None` render without the file-pointer suffix; items with one render with `` — `{path}` `` after the concept.
- [ ] Empty `tech_stack` / `project_ideas` / `concepts_taught` / `prerequisites` arrays render gracefully (either as `_(none)_` or as an omitted section — pick one and be consistent in tests).
- [ ] `telegram.sender.send_document` is called with `filename=f"{owner}-{repo}.md"` and the rendered blob.
- [ ] Filename sanitization handles owner/repo names with dots (`golang.org`-style), dashes, and uppercase letters without breaking; falls back to `{job_id}.md` for pathological cases.
- [ ] Document fires BEFORE the summary text message; if document send fails, summary message still fires and job still completes (verify with mocked `send_document` raising).
- [ ] Tests: rendering with full / minimal / archived / no-README payloads; filename sanitization edge cases.
- [ ] PR demo: paste a repo URL, receive both the `.md` attachment and the summary message in chat.
